## How Can I Contribute?

### Reporting Bugs

Before creating bug reports, please check existing issues to avoid duplicates. When creating a bug report, include:

- **Clear title and description**
- **Steps to reproduce** the issue
- **Expected behavior**
- **Actual behavior**
- **Screenshots** (if applicable)
- **Browser and OS version**

### Suggesting Enhancements

Enhancement suggestions are welcome! Please provide:

- **Clear use case** for the feature
- **Detailed description** of the proposed functionality
- **Mockups or examples** (if applicable)

### Pull Requests

1. Fork the repo and create your branch from `main`
2. Make your changes
3. Test thoroughly across different browsers
4. Update documentation if needed
5. Write clear commit messages
6. Submit your pull request!

## Development Guidelines

### Code Style

- Use meaningful variable and function names
- Add comments for complex logic
- Keep functions small and focused
- Follow existing code formatting

### CSS

- Use CSS custom properties (variables) for colors and common values
- Maintain mobile-first responsive design
- Keep selectors specific but not overly complex
- Group related styles together

### JavaScript

- Write vanilla JavaScript (no frameworks)
- Use modern ES6+ features
- Handle errors gracefully
- Test in multiple browsers

### Commit Messages

- Use present tense ("Add feature" not "Added feature")
- Keep first line under 50 characters
- Be descriptive but concise

## Testing

Before submitting:

- [ ] Test in Chrome, Firefox, Safari
- [ ] Test on mobile devices
- [ ] Verify all timer functions work correctly
- [ ] Check settings save/load properly
- [ ] Ensure responsive design works at all breakpoints

## Questions?

Feel free to open an issue for any questions or clarifications!
